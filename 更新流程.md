# 跑团数据更新流程

## 项目信息
- **仓库**: `https://github.com/Fred-Chan/RunningClubITCCC.git`
- **本地路径**: `./RunningClubITCCC/`
- **主文件**: `index.html`
- **部署**: Vercel 自动部署（GitHub push 后自动同步）

## 更新步骤

### 1. 解析接龙数据
用户发来微信群接龙，格式示例：
```
#接龙 15号跑团5月份跑步接龙
1. 杨靖 0501 5km
2. 徐岚🏅 0501 8.6km
...
```

逐条提取：序号、姓名、日期（MMDD）、距离（数字+单位）。

> **注意**：姓名中的 emoji 装饰（如🏅）不录入；日期保持4位数字格式。

### 2. 对比已有数据，识别新增记录
读取 `index.html` 中 `localData` 数组（搜索 `{ id:` 定位），对比接龙内容：
- 已存在的记录 → 跳过
- 新增的记录 → 追加

对比维度：**姓名 + 日期 + 距离** 三者完全一致视为已有。

### 3. 追加新记录
在 `localData` 数组末尾（最后一个 `}` 之后、`];` 之前）追加新条目，格式：
```javascript
{ id: N, name: '姓名', date: 'MMDD', distance: 数字, unit: 'km' },
```

- `id`: 递增，取上一条 id + 1
- `name`: 去掉 emoji，保留纯文字
- `date`: 4位数字，月日各2位
- `distance`: 纯数字（如 `8.6`，不要写 `8.60` 或 `8.6km`）
- `unit`: 固定 `'km'`

### 4. 检查是否需要新增人员配色
如果新记录的姓名在 `personColors` 对象中不存在，需要添加配色条目：
```javascript
'姓名': '#色值',    // 颜色描述 (色相X°)
```

配色原则：基于 HSL 色相环均匀分布，确保与相邻人员颜色区分清晰。

### 5. Git 提交推送
```bash
cd ./RunningClubITCCC
git add .
git commit -m "feat: 更新X月跑步数据(接龙N)"
git push
```

- commit message 格式：`feat: 更新{月份}跑步数据(接龙{序号})`
- push 后 Vercel 自动部署，无需手动操作

### 6. 通知用户
告知更新完成，简要说明新增了哪些记录。

## 常见问题

| 问题 | 处理方式 |
|------|---------|
| 同一人同一天跑了多次 | 正常录入，每条独立 id |
| 距离带小数点（如 5.10km） | 录入为 `5.1`，去掉末尾无效零 |
| 新人首次出现 | 录入数据 + 添加 `personColors` 配色 |
| 接龙中重复已有数据 | 跳过，不重复录入 |
| Git push 失败 | 先 `git pull --rebase` 再 push |
